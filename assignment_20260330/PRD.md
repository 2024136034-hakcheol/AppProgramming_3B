# PRD: 개인 할 일 관리 앱 (Todo Manager)

## 1. 개요

### 1.1 목적
매일 10~20개의 할 일을 효율적으로 관리하기 위한 개인용 웹 애플리케이션

### 1.2 목표 사용자
- 혼자 사용하는 개인 사용자
- 업무/개인/공부 영역을 구분해서 할 일을 관리하고 싶은 사람

### 1.3 기술 스택
- HTML, CSS, Vanilla JavaScript (순수 자바스크립트)
- LocalStorage (브라우저 내 데이터 영구 저장)
- 외부 라이브러리 없음 (의존성 제로)

---

## 2. 기능 요구사항

### 2.1 할 일 추가
- 텍스트 입력 후 Enter 키 또는 추가 버튼으로 등록
- 등록 시 카테고리 선택 필수 (업무 / 개인 / 공부)
- 입력값 공백 검증 (빈 값 등록 불가)

### 2.2 할 일 수정
- 기존 항목 더블클릭 또는 수정 버튼으로 인라인 편집
- 텍스트 및 카테고리 모두 수정 가능
- 수정 완료: Enter 키 또는 저장 버튼
- 수정 취소: Esc 키 또는 취소 버튼

### 2.3 할 일 삭제
- 항목별 삭제 버튼
- 완료된 항목 전체 삭제 버튼 (일괄 정리)

### 2.4 완료 체크
- 체크박스 클릭으로 완료/미완료 토글
- 완료된 항목은 텍스트에 취소선 표시 및 시각적 구분
- 완료 항목은 목록 하단으로 자동 이동

### 2.5 카테고리 분류
| 카테고리 | 설명 |
|---------|------|
| 업무 | 직장/프로젝트 관련 할 일 |
| 개인 | 일상/생활 관련 할 일 |
| 공부 | 학습/자기계발 관련 할 일 |

- 카테고리별 필터링 탭 제공 (전체 / 업무 / 개인 / 공부)
- 카테고리별 색상 배지로 구분

### 2.6 진행률 보기
- 상단에 전체 진행률 프로그레스 바 표시
  - `완료된 항목 수 / 전체 항목 수 × 100%`
- 숫자로도 표시: `3 / 10 완료 (30%)`
- 카테고리 필터 적용 시 해당 카테고리의 진행률로 업데이트

---

## 3. 비기능 요구사항

### 3.1 데이터 영속성
- 모든 데이터는 브라우저 LocalStorage에 저장
- 페이지 새로고침, 브라우저 재시작 후에도 데이터 유지
- 저장 시점: 추가/수정/삭제/체크 변경 시 즉시 자동 저장

### 3.2 UX
- 별도 설치 없이 브라우저에서 바로 실행 (단일 HTML 파일)
- 모바일/데스크탑 모두 사용 가능한 반응형 레이아웃
- 할 일이 없을 때 빈 상태 안내 메시지 표시

## 7. 개발 범위 외 (Out of Scope)

- 서버, 백엔드, DB 없음
- 사용자 로그인/회원가입 없음
- 알림/리마인더 없음
- 할 일 순서 드래그 정렬 없음
- 다중 기기 동기화 없음
