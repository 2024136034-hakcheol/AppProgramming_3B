# 추가 기능 프롬프트 — 우선순위(Priority) 기능

## 기능 설명

할 일에 **높음 / 보통 / 낮음** 3단계 우선순위를 부여하여
무엇을 먼저 해야 하는지 한눈에 파악할 수 있도록 한다.

---

## 작성한 프롬프트

```
index.html에 우선순위(Priority) 기능을 추가해 줘.

[데이터]
- 기존 todo 객체에 priority 필드 추가: "높음" | "보통" | "낮음"
- 기본값은 "보통"

[입력 폼]
- 카테고리 드롭다운 옆에 우선순위 드롭다운(priority-select)을 추가
  - 옵션: 🔴 높음 / 🟡 보통 / 🟢 낮음

[목록 표시]
- 각 할 일 항목에 우선순위 배지(pri-badge)를 카테고리 배지 옆에 표시
- 항목 왼쪽에 우선순위별 색상 테두리 표시:
    높음 → 빨간색(#fc8181)
    보통 → 주황색(#f6ad55)
    낮음 → 초록색(#68d391)

[정렬]
- 미완료 항목을 먼저, 완료 항목을 나중에 표시
- 같은 완료 상태 안에서는 높음 → 보통 → 낮음 순으로 정렬

[수정]
- 인라인 수정 모드에서 우선순위도 변경 가능하도록 <select> 추가

[저장]
- priority 값도 localStorage에 함께 저장/복원
```

---

## 구현 결과 확인 포인트

- [ ] 할 일 추가 시 우선순위 선택 가능
- [ ] 항목 왼쪽 테두리 색이 우선순위에 따라 다르게 표시
- [ ] 높음 우선순위 항목이 목록 최상단에 표시
- [ ] 수정 모드에서 우선순위 변경 가능
- [ ] 새로고침 후 우선순위 값이 유지됨
