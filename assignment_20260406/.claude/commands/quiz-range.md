quiz-data.js 파일을 읽고, $ARGUMENTS번째 문제부터 $ARGUMENTS번째 문제까지를 검토해 줘.

## 인수 파싱
$ARGUMENTS는 공백으로 구분된 두 숫자야.
- 첫 번째 숫자 = 시작 번호 (1-based)
- 두 번째 숫자 = 끝 번호 (1-based, 포함)
- 예) `/quiz-range 1 10` → QUESTIONS 배열의 1번~10번 문제 검토
- 예) `/quiz-range 11 20` → QUESTIONS 배열의 11번~20번 문제 검토
- 인수가 없거나 잘못된 경우: "사용법: /quiz-range <시작번호> <끝번호>" 안내 후 종료

## 검토 항목 (문제마다 아래 5가지를 모두 확인)
1. **정답 유일성** — 정답(answer 인덱스)이 하나뿐인가? 다른 선택지도 정답이 될 수 있지 않은가?
2. **최상급 표현** — `가장`, `최초`, `최대`, `최소`, `최고`, `최저`, `유일`, `처음`, `첫`, `제일` 포함 시 측정 기준이 명시되어 있는가?
3. **시간·범위 명확성** — 변할 수 있는 정보(인구, 순위 등)에 시점 기준이 있는가?
4. **선택지 일관성** — 4개 선택지의 형식(단위, 표기)이 통일되어 있는가?
5. **해설 정확성** — explanation이 question·answer와 일치하는가?

## 출력 형식

### 검토 범위: $ARGUMENTS번 문제

각 문제를 아래 형식으로 출력해:

---
**[번호] ID — 카테고리 (difficulty)**
> 문제: …
> 정답: 선택지 텍스트

| 항목 | 판정 | 비고 |
|------|------|------|
| 정답 유일성 | ✅ / ⚠️ | |
| 최상급 표현 | ✅ / ⚠️ / — | — 는 해당 없음 |
| 시간·범위 | ✅ / ⚠️ / — | |
| 선택지 일관성 | ✅ / ⚠️ | |
| 해설 정확성 | ✅ / ⚠️ | |

---

## 최종 요약
검토가 끝나면 아래 요약을 출력해:

- 검토 문제 수: N개
- 이상 없음: N개
- 수정 권고: N개 → 해당 ID 목록 나열

수정 권고 문제가 있으면 수정안을 함께 제안해 줘.
