quiz-data.js 파일을 읽고, 새 퀴즈 문제를 한 개 생성해서 추가해 줘.

## 인수 파싱
$ARGUMENTS는 공백으로 구분된 두 값이야.
- 첫 번째 값 = 카테고리 (`한국사` | `과학` | `지리` | `일반상식`)
- 두 번째 값 = 난이도 (`easy` | `medium` | `hard`)
- 예) `/quiz-add 한국사 medium` → 한국사 카테고리 medium 난이도 문제 1개 추가
- 인수가 없거나 허용 값이 아닌 경우: "사용법: /quiz-add <카테고리> <난이도>" 안내 후 종료

## 1단계 — 파일 읽기 및 현황 파악

quiz-data.js를 읽고 다음을 확인해:
1. 지정 카테고리의 기존 문제 목록과 마지막 ID 번호
2. ID 접두어 규칙: `한국사`→`KH`, `과학`→`SC`, `지리`→`GE`, `일반상식`→`GK`
3. 새 ID = 접두어 + `-` + (마지막 번호 + 1, 세 자리 패딩) 예) `KH-011`
4. 기존 문제와 **주제가 겹치지 않도록** 기존 question 목록을 훑어봐

## 2단계 — 문제 초안 작성

아래 스키마에 맞춰 문제 1개를 작성해:

```json
{
  "id": "XX-NNN",
  "category": "카테고리",
  "difficulty": "easy|medium|hard",
  "question": "문제 텍스트",
  "choices": ["①", "②", "③", "④"],
  "answer": 0,
  "explanation": "해설 텍스트"
}
```

작성 규칙:
- 선택지는 4개, 정답은 answer 인덱스(0~3)로 지정
- 선택지 형식(단위, 표기)을 4개 모두 통일
- 명백히 틀린 오답 3개를 섞어 혼란을 방지
- 해설은 정답 근거와 오답과의 차이를 한 문장 이상으로 설명

## 3단계 — 검증 (5개 항목 모두 통과해야 추가 가능)

| 항목 | 기준 |
|------|------|
| 정답 유일성 | 정답이 하나뿐인가? 다른 선택지도 정답이 될 여지가 없는가? |
| 최상급 표현 | `가장`, `최초`, `최대`, `최소`, `최고`, `최저`, `유일`, `처음`, `첫`, `제일` 포함 시 측정 기준 명시 |
| 시간·범위 명확성 | 변할 수 있는 정보(인구·순위·현황 등)에 시점 기준 명시 (예: "2024년 기준") |
| 선택지 일관성 | 4개 선택지의 형식·단위·표기가 통일되어 있는가? |
| 해설 정확성 | explanation이 question·answer와 일치하고 사실관계가 정확한가? |

⚠️ 항목이 하나라도 실패하면 문제를 수정한 뒤 다시 검증해. 모두 통과할 때까지 반복.

> 정답: [인덱스번호] — [선택지 텍스트]
> 해설: …
